================================================================================
ヒント1: 攻撃の方向性
================================================================================

【脆弱性の理解】

vulnerable_server.py を読むと、以下の脆弱性が見つかります：

1. orphan_responses というグローバルリストがある
   - すべてのクライアントで共有されている
   - 送信失敗したレスポンスがここに保存される

2. get_orphans というメソッドがある
   - orphan_responses の中身を返してくれる
   - 認証チェックがない？

3. get_secret は遅延を指定できる
   - delay_ms パラメーターで処理を遅らせられる
   - 遅延中にクライアントが切断したらどうなる？


【攻撃の流れを考える】

Q: admin の秘密を直接取得できないなら、どうやって取得する？
A: admin が自分の秘密を取得 → 何らかの理由でレスポンスを受け取れない
   → orphan として残る → guest がそれを取得

Q: どうやって admin のレスポンスを orphan にする？
A: admin が get_secret を呼ぶ → 遅延中に admin が切断
   → レスポンス送信失敗 → orphan_responses に保存


【次のステップ】

1. wait_time を適切に設定する
   - admin の遅延（2000ms）より長く待つ必要がある

2. orphan を取得するメソッドを呼び出す
   - どのメソッドが orphan を返す？

まだ分からなければ hint2.txt を見てください。
